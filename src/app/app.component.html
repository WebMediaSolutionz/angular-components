Buttons

There are two basic categories of button-like objects: buttons and anchors. Both are based on the existing <button> and <a> components in your browser; they are invoked by adding the custom property pc-button or pc-flat-button for a flat (borderless) button.

Usage

The <button> tag should be used when performing an action on the current view; the <a> tag should be used when navigating to another view.

Cards

Cards describe a related set of information or controls, usually with a heading or footer (or both), and sometimes with a set of actions that can be performed, such as the submission of data entered into a form in the card content.

Cards are distinct from widgets, which are discrete functionality but typically *do not* have heading/footer with the traditional action buttons at the bottom of the widget. See widget documentation for details.
